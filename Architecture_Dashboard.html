System Architecture

Predictive Intelligence Pipeline

An AI-Powered Early Warning System Using Hybrid Data Sources for Landslide Disaster Prevention.

Phase 1: Hybrid Data Acquisition

Historical Datasets

Kaggle & Ground Data

Used for baseline temporal training. Includes Rainfall volume, Soil Saturation, Slope Angle, and Seismic activity mappings.

Sentinel Satellites

Copernicus / Earth Engine

Real-time Raster feeds fusing Sentinel-1 SAR (cloud-penetrating radar) and Sentinel-2 Optical imagery for visual deformation.

Meteorological Data

IMD / Live Weather APIs

Continuous fetching of rainfall telemetry pipelines to feed the predictive ML engine with live risk events.

Phase 2: Dual AI Modeling Engine

Spatial Detection (YOLO)

PyTorch / TensorFlow

Processes satellite rasters using YOLO object detection to draw bounding boxes around active landslides instantly. Replaces slow pixel-wise segmentation.

Temporal Prediction (ML)

Scikit-Learn / XGBoost

Analyzes real-time rainfall and historical tabular data to forecast landslide probability BEFORE they actually happen visually.
Backend Fusion API: Python FastAPI processes both modules simultaneously on local hardware or cloud GPU instance.

Phase 3: Alert & Dissemination

Vercel React.js Dashboard

End-User Interface & Mapbox Rendering

The final interactive GUI deployed on Vercel. Authorities (e.g., NDMA) log in to view a live hazard map layered with Red (Critical Warning), Orange (Watch), and Green (Safe) regional bounding boxes.
